# Contributing

## Development

1. Install dependencies using the detected package manager.
2. Run the development server.
3. Before opening a pull request, run the available verification commands.

## Pull Requests

PRs should be small, focused, and easy to review.

Include:

- what changed
- why it changed
- how it was tested
- risks or follow-up work

## AI-Assisted Changes

AI-generated changes must follow `AGENTS.md` and must pass the same checks as human-written changes.
