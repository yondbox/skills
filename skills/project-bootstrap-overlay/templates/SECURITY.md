# Security Policy

## Reporting a Vulnerability

If you discover a vulnerability, please report it privately to the repository owner.
Do not disclose vulnerabilities publicly until they have been reviewed.

## Secrets

Do not commit secrets, tokens, private keys, or real environment files.
Use `.env.example` to document required variables without real values.

## Dependency Security

Dependencies should be reviewed and updated regularly through Dependabot, Renovate, or an equivalent process.
