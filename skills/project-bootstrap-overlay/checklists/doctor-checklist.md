# Doctor Checklist

Use this checklist during the Doctor phase.

## Repository

- [ ] Is this a git repository?
- [ ] What is the repository name?
- [ ] Is README present?
- [ ] Is LICENSE present?
- [ ] Is docs/ present?
- [ ] Is this likely generated by an official generator?
- [ ] Is this likely an instructor template?

## Stack

- [ ] Detect language(s)
- [ ] Detect framework(s)
- [ ] Detect runtime version
- [ ] Detect package manager
- [ ] Detect lockfiles
- [ ] Detect monorepo tooling

## Local Development

- [ ] Install command
- [ ] Dev command
- [ ] Build command
- [ ] Environment variable files
- [ ] .env.example
- [ ] Docker / Dev Container / compose

## Quality

- [ ] Formatter
- [ ] Linter
- [ ] Type checker
- [ ] Test runner
- [ ] E2E framework
- [ ] Coverage

## CI/CD

- [ ] CI provider
- [ ] Workflow files
- [ ] Build in CI
- [ ] Test in CI
- [ ] Deployment workflow
- [ ] Preview deploy
- [ ] Production deploy

## Security

- [ ] Third-party actions
